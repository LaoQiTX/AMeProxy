# 代理工具实现原理文档

## 一、项目概述

本文档基于 [Clash Nyanpasu](https://github.com/IdleIronMan/clash-nyanpasu) 项目，分析代理工具的核心实现原理，帮助你理解并开发类似的代理客户端。

**Clash Nyanpasu** 是一个基于 **Tauri** 框架开发的 Clash GUI 客户端，主要特点：
- 支持多种 Clash 核心（Mihomo、Clash Premium、Clash Rust）
- 支持 TUN 模式（系统级代理）
- 支持 PAC 自动代理配置
- 支持 JavaScript/Lua 脚本增强配置

## 三、代理协议实现原理

### 3.1 混合代理端口 (Mixed Port)

Mixed Port 是 Clash 的核心特性，允许在单一端口同时处理 HTTP、HTTPS、SOCKS5 协议请求。

**配置示例：**
```yaml
mixed-port: 7890          # 混合代理端口
allow-lan: false         # 是否允许局域网连接
mode: rule               # 规则模式 (rule/global/rules)
log-level: info          # 日志级别
```

**工作原理：**
```
客户端请求 ──▶ 7890端口 ──┬──▶ HTTP 解析 ──▶ 规则匹配
                         ├──▶ SOCKS5 解析 ──▶ 规则匹配
                         └──▶ HTTPS 解析 ──▶ 规则匹配 (CONNECT)
```

### 3.2 支持的代理协议

Clash 核心原生支持以下代理协议：

| 协议 | 配置类型 | 说明 |
|------|---------|------|
| Shadowsocks | `ss` | 基于 AES/Salty20 等加密的代理协议 |
| VMess | `vmess` | V2Ray 核心协议，支持 UUID 认证 |
| Trojan | `trojan` | 基于 TLS 的代理协议 |
| VLESS | `vless` | 无加密的轻量协议，需配合 TLS |
| Hysteria | `hysteria` | 基于 UDP 的拥塞控制协议 |
| WireGuard | `wireguard` | 现代 VPN 协议 |
| Snell | `snell` | Snell 协议 |

**节点配置示例：**
```yaml
proxies:
  # Shadowsocks
  - name: "SS节点"
    type: ss
    server: 1.2.3.4
    port: 8388
    cipher: aes-256-gcm
    password: password123

  # VMess
  - name: "VMess节点"
    type: vmess
    server: 5.6.7.8
    port: 10086
    uuid: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
    alterId: 0
    cipher: auto

  # Trojan
  - name: "Trojan节点"
    type: trojan
    server: 9.9.9.9
    port: 443
    password: password123
    sni: example.com
```

## 四、代理数据流详解

### 4.1 标准代理模式数据流

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│   客户端    │────────▶│  Clash      │────────▶│  远程服务器  │
│ (浏览器/APP)│         │  Core       │         │ (google.com)│
└─────────────┘◀────────│  :7890      │◀────────└─────────────┘
                        └─────────────┘
                              │
                              ▼
                        ┌─────────────┐
                        │  规则引擎    │
                        │  匹配处理    │
                        └─────────────┘
```

**处理流程：**
1. **请求接收**：Clash 在 7890 端口接收代理请求
2. **协议解析**：识别 HTTP/SOCKS5/TCP 请求类型
3. **规则匹配**：根据配置的规则决定出站策略
4. **代理连接**：通过选定的代理协议连接远程服务器
5. **数据转发**：双向转发客户端与远程服务器的数据
6. **响应返回**：将远程服务器响应返回给客户端

### 4.2 TUN 模式数据流

TUN 模式是系统级代理，直接在网络层接管所有流量。

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│   应用     │────────▶│   TUN       │────────▶│  Clash      │
│  (全部流量) │         │   网卡      │         │  Core       │
└─────────────┘◀────────│  (虚拟)     │◀────────└─────────────┘
                        └─────────────┘
                              │
                              ▼
                        ┌─────────────┐
                        │   DNS劫持    │
                        │  (Fake-IP)   │
                        └─────────────┘
```

**TUN 配置：**
```yaml
dns:
  enable: true
  listen: 0.0.0.0:53
  enhanced-mode: fake-ip
  fake-ip-range: 198.18.0.1/16

tun:
  enable: true
  stack: system          # system/gvisor/mixed
  dns-hijack:
    - any:53
  auto-route: true
  auto-detect-interface: true
```

### 4.3 PAC 代理模式数据流

PAC (Proxy Auto-Config) 根据规则自动选择代理或直连。

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│   浏览器    │────────▶│   PAC文件    │────────▶│  规则匹配   │
│             │         │  (JS函数)    │         │  返回代理   │
└─────────────┘◀────────│             │◀────────│   或 DIRECT │
                        └─────────────┘         └─────────────┘
```

## 五、规则系统实现

### 5.1 规则类型

| 规则类型 | 示例 | 说明 |
|---------|------|------|
| DOMAIN | `DOMAIN,google.com` | 精确域名匹配 |
| DOMAIN-SUFFIX | `DOMAIN-SUFFIX,cn` | 域名后缀匹配 |
| DOMAIN-KEYWORD | `DOMAIN-KEYWORD,google` | 域名关键词匹配 |
| GEOIP | `GEOIP,CN` | IP地理位置匹配 |
| IP-CIDR | `IP-CIDR,192.168.0.0/16` | IP段匹配 |
| RULE-SET | `RULE-SET,geolocation-cn` | 规则集匹配 |
| PROCESS-NAME | `PROCESS-NAME,chrome.exe` | 进程名匹配 |
| DOMAIN-SUFFIX,ip-api.com | 无 | DNS 黑名单 |

### 5.2 规则匹配流程

```rust
fn match_rule(request: &Request) -> MatchResult {
    for rule in rules {
        if rule.match(&request) {
            return MatchResult {
                policy: rule.policy,
                proxy: select_proxy_from_group(rule.policy)?
            };
        }
    }
    return MatchResult { policy: DIRECT };
}
```

## 六、代理组管理

### 6.1 代理组类型

| 类型 | 说明 | 示例 |
|------|------|------|
| url-test | 自动选择最快节点 | 测试 URL 延迟 |
| fallback | 故障转移 | 节点不可用时切换 |
| load-balance | 负载均衡 | 轮询分配请求 |
| select | 手动选择 | 用户手动选择 |
| relay | 链式代理 | 节点A→节点B |

### 6.2 代理组配置

```yaml
proxy-groups:
  # 手动选择代理组
  - name: Proxy
    type: select
    proxies:
      - SS节点
      - VMess节点
      - Trojan节点
      - 自动选择

  # 自动选择最快节点
  - name: 自动选择
    type: url-test
    proxies:
      - SS节点
      - VMess节点
    url: http://www.gstatic.com/generate_204
    interval: 300

  # 故障转移
  - name: 故障转移
    type: fallback
    proxies:
      - SS节点
      - VMess节点
    url: http://www.gstatic.com/generate_204
    interval: 300

  # 负载均衡
  - name: 负载均衡
    type: load-balance
    proxies:
      - SS节点
      - VMess节点
    url: http://www.gstatic.com/generate_204
    interval: 300
```

## 七、核心管理实现

### 7.1 Clash 核心启动流程

```rust
pub async fn run_core() -> Result<()> {
    // 1. 加载配置
    let config = load_config()?;
    
    // 2. 生成运行时配置
    let runtime_config = generate_runtime_config(config)?;
    
    // 3. 启动核心进程
    let mut command = Command::new("mihomo");
    command.args(&["-d", "."]);
    
    // 4. 写入配置文件
    let config_path = write_config_file(runtime_config)?;
    command.arg("-f").arg(config_path);
    
    // 5. 启动进程
    let child = command.spawn()?;
    
    // 6. 等待服务就绪
    wait_for_port(7890).await?;
    
    Ok(())
}
```

### 7.2 REST API 控制器

Clash 提供 REST API 用于外部控制：

```yaml
# Clash 配置
external-controller: 127.0.0.1:9090
# external-ui: ./ui    # 可选：提供 Web UI
```

**API 端点：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /proxies | 获取所有代理和代理组 |
| GET | /proxies/{name} | 获取特定代理详情 |
| PUT | /proxies/{group} | 更新代理组选中的代理 |
| GET | /configs | 获取当前配置 |
| PUT | /configs | 更新配置 |
| GET | /rules | 获取规则列表 |
| GET | /traffic?direct=1 | 获取流量统计 |
| GET | /connections | 获取连接列表 |

**API 调用示例：**
```rust
// 获取代理列表
async fn get_proxies() -> Result<Vec<Proxy>> {
    let resp = reqwest::get("http://127.0.0.1:9090/proxies").await?;
    let data: Value = resp.json().await?;
    Ok(data["proxies"].as_array().unwrap())
}

// 更新代理选择
async fn select_proxy(group: &str, proxy: &str) -> Result<()> {
    let client = reqwest::Client::new();
    client.put(&format!("http://127.0.0.1:9090/proxies/{}", group))
        .json(&json!({"name": proxy}))
        .send()
        .await?;
}
```

### 7.3 WebSocket 实时监控

```rust
use tokio_tungstenite::{connect_async, ws_stream};
use futures_util::{SinkExt, StreamExt};

async fn connect_websocket() -> Result<()> {
    let (ws_stream, _) = connect_async(
        "ws://127.0.0.1:9090/connections"
    ).await?;
    
    let (mut write, mut read) = ws_stream.split();
    
    while let Some(msg) = read.next().await {
        match msg? {
            Message::Text(text) => {
                let data: ConnectionData = serde_json::from_str(&text)?;
                update_traffic_stats(data);
            }
            _ => {}
        }
    }
    Ok(())
}
```

## 八、系统集成

### 8.1 系统代理设置

**Windows (WinAPI)：**
```rust
#[repr(C)]
struct INTERNET_PROXY_INFO {
    dwAccessType: DWORD,
    lpszProxy: LPCWSTR,
    lpszProxyBypass: LPCWSTR,
}

fn set_system_proxy(host: &str, port: u16) -> Result<()> {
    unsafe {
        let proxy = format!("http={}:{}", host, port);
        let proxy_info = INTERNET_PROXY_INFO {
            dwAccessType: INTERNET_OPEN_TYPE_PROXY,
            lpszProxy: to_wide_string(&proxy),
            lpszProxyBypass: to_wide_string("localhost;127.*;10.*;172.16.*;192.168.*"),
        };
        
        InternetSetOptionW(
            None,
            INTERNET_OPTION_PROXY,
            Some(&proxy_info as *const _ as *const _),
            std::mem::size_of::<INTERNET_PROXY_INFO>() as DWORD
        )?;
    }
    Ok(())
}
```

**macOS (SystemConfiguration)：**
```rust
fn set_macos_proxy(host: &str, port: u16) -> Result<()> {
    let mut session = SCDynamicStoreCore::new(None)?;
    let proxy_settings = hashmap! {
        "HTTPEnable" => 1,
        "HTTPProxy" => host,
        "HTTPPort" => port,
        "HTTPSEnable" => 1,
        "HTTPSProxy" => host,
        "HTTPSPort" => port,
    };
    session.set_multiple(proxy_settings)?;
    Ok(())
}
```

## 十、安全注意事项

1. **不存储明文密码**：敏感配置应加密存储
2. **验证节点来源**：检查订阅内容的完整性
3. **隔离网络请求**：防止 DNS 泄漏
4. **定期更新核心**：修复安全漏洞
5. **最小权限原则**：TUN 模式下只接管必要流量

## 十二、总结

代理工具的核心实现依赖于成熟的代理协议库（如 Clash）。开发重点在于：

1. **用户界面**：提供友好的配置和管理界面
2. **核心管理**：高效管理底层代理进程
3. **系统集成**：实现系统级代理设置
4. **规则系统**：灵活配置流量分流规则
5. **状态监控**：实时展示连接和流量状态

Clash Nyanpasu 作为一个优秀的开源项目，展示了如何将 Clash 核心封装成现代化的桌面应用，其架构和实现方式值得学习和借鉴。
